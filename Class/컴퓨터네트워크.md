##### Computer Network(Computer + Net + Work)
- 유무선 통신선에 컴퓨터들이 연결되어 데이터를 서로 주고 받으면 작업 할 수 있는 형태
- Network의 한자어 = 망 (ex. 컴퓨터 망, 컴퓨터 통신망)
##### Internet(Inter+Network)
- 수 많은 컴퓨터 네트워크들이 TCP/IP(Transmission Control Protocol, Internet Protocol) 프로토콜 기반으로 전세계에 걸쳐 상호 연결된(Interconnected) 컴퓨터 네트워크
- Internet은 특정 통신 프로토콜들을 사용하는 특정 네트워크를 지칭하는 고유명사
- 즉, 인터넷 외에도 많은 다른 네트워크들이 존재
##### Computer Communication Protocol
- 컴퓨터 및 통신 장비 간의 데이터(메시지) 통신을 위해 설계된 일련의 규약
- 컴퓨터나 통신 장비 HW 또는 SW에 의해 실행됨

#### 기초용어
##### Analog & Digital
- 컴퓨터는 디지털로 표현되기 때문에 아날로그 음성 및 영상의 디지털 변환, 저장, 전송되어야 함
##### Analog
- 어떤 양이나 데이터를 연속적으로 변화하는 물리량(전압, 전류 등)으로 표현
- 연속적(Continuous)인 선의 형태로 정보 전달
##### Digital
- 어떤 양이나 데이터를 2진수 값으로 표현
- 이산적(Discrete)인 형태로 정보 전달
##### 컴퓨터
- 아날로그 컴퓨터: 아날로그 물리정보를 입력 받아 처리
- 디지털 컴퓨터: 2진 디지털 정보를 입력 받아 처리
- 하이브리드 컴퓨터: 아날로그와 디지털 컴퓨터의 장점을 혼합
##### 디지털 컴퓨터 구조
- CPU(ALU(Arithmetic Logic Unit)+CU(Control Unit)) + Meomry + IO(입출력) 장치가 버스를 통해 연결됨
- von Neumann 전통적인 구조, Harvard Computer Architecture 개량된 구조
- Register: 컴퓨터 명령어를 일반메모리에 저장하면 속도가 느리기때문에 내부에 register 메모리로 CPU 연산속도를 빠르게함
##### Computer Networking
- 컴퓨터 IO 부분에 연결 접점이 되는 네트워크 인터페이스(ex. LAN 카드)와 이를 통해 연결되는 wired/wireless(유/무선) 전송매체를 통해 컴퓨터들이 통신할 수 있도록 연결하는 것
##### 공식표준화기구(De-Jure)
- 표준화 기구(ISO, IEEE, ANSI, IETF)

##### 네트워크
- 통신 매체를 매개로 서로 연결되어 데이터를 교환하는 시스템의 모음
- 프로토콜을 사용하여 데이터를 교환하는 시스템의 집합
- 네트워크끼리는 라우터로 연결됨

##### 인터페이스
- 시스템끼리 연결하기 위한 표준화된 접근 방법
- 시스템과 전송매체의 연결 지점에 대한 규격
- 두 시스템이 연동하기 위한 특정한 접촉 지점
- 시스템과 동등관계 또는 상하관계
##### 표준화
- 프로토콜처럼 서로 다른 시스템을 상호 연동해 동작시키기 위한 표준화라는 연동 형식의 통일필요
- ex) 프린터 - A4용지
##### 프로토콜
- 상호 연동되는 시스템이 데이터를 교환할때 표준화된 특정 규칙인 프로토콜을 따름
- 정보의 형식, 절차적 순서 등에 초점
- 시스템과 동등관계
##### 인터넷 
- 전 세계의 모든 네트워크가 유기적으로 연결되어 동작하는 통합 네트워크
- 데이터 전달기능에선 공통으로 IP프로토콜을 사용
- 인터넷은 IP(Internet Protocol)의 첫단어에서 유래

##### 진법(notation)
- 디지털 컴퓨터는 2진수 기반으로 숫자를 표현하기 위해 많은 자릿수를 차지하고, 10진수에 비해 사람이 인식하기 어렵기 때문에 2의 승수이면서 10진수와 가까운 8진수와 16진수를 컴퓨터에서 많이 사용
- 2진수: 2개(0, 1)의 숫자로 표현
- 8진수: 8개(0~7)의 숫자로 표현
- 10진수: 10개(0~9)의 숫자로 표현
- 16진수: 16개(0~9, A, B, C, D, E, F)의 숫자로 표현
자릿수 ex) 259 = 2x10^2 + 5x10^1 + 9x10^0

##### 진법 변환
30(8진수) = 3x8^1 + 0x8^0 = 24(10진수)
54(16진수) = 5x16^1 + 4x16^0 = 84(10진수)

##### BCD코드(Binary coded decimal)
- 십진수 한 자리를 4비트의 2진수로 표기
- 9(1001)가 최대
- ex) (0001 0000)BCD = (10)10

##### BCD덧셈
- (1010 ~1111) 6개의 숫자가 남기 때문에 2진수 덧셈에 보정이 필요
- 2진수 합이 1010보다 크거나 같으면 0110(6)을 더해줌
- ex) 4+5 = 9 -> 0100 + 0101 =1001
-  4+8 = 12 -> 0100 + 1000 = 1100 + 0110(6) = (1 0010)BCD
##### 비트 표현 개수
- 4비트로 나타낼 수 있는 최대숫자 개수 2^4 = 16 (0~15)
- 8비트로 나타낼 수 있는 최대숫자 개수 2^8 = 256 (0~255)
- 16비트로 나타낼 수 있는 최대숫자 개수 2^16 = 65536 (0~65535)
- 1바이트 = 256개의 숫자 표현 (0~255)
- nible = 4 bit 
- byte = 8 bit 
- word = 2 byte
- quad word = 4 word = 8 byte
- octo word = 8 word = 16 byte

##### 보수(Complement)
- 컴퓨터는 2진수로 표현되기 때문에 음수를 보수를 이용해 표현
- 보수는 보충을 해주는 수
ex) 1에 대한 10의 보수는 9
4에 대한 16의 보수는 12

#### 보수를 이용한 음의 정수 표현
##### 부호 절대값
- 최상의 비트(MSB(Most Significant Bit))로 음수 표현
- 0이 양수와 음수로 나누어지고 덧셈, 뺄셈이 매우 불편(큰수가 위로오게하고 부호 설정)
- -(2^(n-1)-1)~(2^(n-1)-1) , 8비트 -127 ~ 127
ex) 0000 000 = +0
1000 0000 = -0
##### 1의 보수
- 2진수를 각 자리수의 값이 1인 수를 빼면 1의 보수
- 0이 양수와 음수로 존재
- -(2^(n-1)-1)~(2^(n-1)-1) \, 8비트 -127 ~ 127
ex) 11111111 - 00000011(3) = 11111100(-3)
##### 1의 보수 뺄셈(추가)
- 빼는 수대신 빼는 수의 1의 보수를 더한다음 자리올림이 생기면 최상의 비트를 제외하고 최하위 비트에 +1, 자리올림이 없으면 1의 보수를 구한 후 -부호를 붙임
ex) 1101 -1001 = 1101+0110 = 10011 -> 0100
0101 -1001 = 0101 + 0110 = 1011 -> -0100
##### 2의 보수
- 1의 보수에서 1을 더한 값
- 0 음수 존재x
- 컴퓨터에서 사용
-  -2^(n-1) ~ (2^(n-1) - 1) ,  8비트 -128 ~ 127
ex) 11111111 - 00000011(3) = 11111100 +1 = 111111101(-3)
##### 2의 보수 뺄셈(추가)
- 빼는 수의 2의 보수를 구하고 더한다음 자리올림이 생기면 자리올림을 제외한 나머지 부분, 생기지않으면 연산결과의 2의보수를 구한 후 -부호
