# 컴퓨터 네트워크


##### Computer Network(Computer + Net + Work)
- 유무선 통신선에 컴퓨터들이 연결되어 데이터를 서로 주고 받으면 작업 할 수 있는 형태
- Network의 한자어 = 망 (ex. 컴퓨터 망, 컴퓨터 통신망)
##### Internet(Inter+Network)
- 수 많은 컴퓨터 네트워크들이 TCP/IP(Transmission Control Protocol, Internet Protocol) 프로토콜 기반으로 전세계에 걸쳐 상호 연결된(Interconnected) 컴퓨터 네트워크
- Internet은 특정 통신 프로토콜들을 사용하는 특정 네트워크를 지칭하는 고유명사
- 즉, 인터넷 외에도 많은 다른 네트워크들이 존재
##### Computer Communication Protocol
- 컴퓨터 및 통신 장비 간의 데이터(메시지) 통신을 위해 설계된 일련의 규약
- 컴퓨터나 통신 장비 HW 또는 SW에 의해 실행됨

#### 기초용어
##### Analog & Digital
- 컴퓨터는 디지털로 표현되기 때문에 아날로그 음성 및 영상의 디지털 변환, 저장, 전송되어야 함
##### Analog
- 어떤 양이나 데이터를 연속적으로 변화하는 물리량(전압, 전류 등)으로 표현
- 연속적(Continuous)인 선의 형태로 정보 전달
##### Digital
- 어떤 양이나 데이터를 2진수 값으로 표현
- 이산적(Discrete)인 형태로 정보 전달
##### 컴퓨터
- 아날로그 컴퓨터: 아날로그 물리정보를 입력 받아 처리
- 디지털 컴퓨터: 2진 디지털 정보를 입력 받아 처리
- 하이브리드 컴퓨터: 아날로그와 디지털 컴퓨터의 장점을 혼합
##### 디지털 컴퓨터 구조
- CPU(ALU(Arithmetic Logic Unit)+CU(Control Unit)) + Meomry + IO(입출력) 장치가 버스를 통해 연결됨
- von Neumann 전통적인 구조, Harvard Computer Architecture 개량된 구조
- Register: 컴퓨터 명령어를 일반메모리에 저장하면 속도가 느리기때문에 내부에 register 메모리로 CPU 연산속도를 빠르게함
##### Computer Networking
- 컴퓨터 IO 부분에 연결 접점이 되는 네트워크 인터페이스(ex. LAN 카드)와 이를 통해 연결되는 wired/wireless(유/무선) 전송매체를 통해 컴퓨터들이 통신할 수 있도록 연결하는 것
##### 공식표준화기구(De-Jure)
- 표준화 기구(ISO, IEEE, ANSI, IETF)

##### 네트워크
- 통신 매체를 매개로 서로 연결되어 데이터를 교환하는 시스템의 모음
- 프로토콜을 사용하여 데이터를 교환하는 시스템의 집합
- 네트워크끼리는 라우터로 연결됨

##### 인터페이스
- 시스템끼리 연결하기 위한 표준화된 접근 방법
- 시스템과 전송매체의 연결 지점에 대한 규격
- 두 시스템이 연동하기 위한 특정한 접촉 지점
- 시스템과 동등관계 또는 상하관계
##### 표준화
- 프로토콜처럼 서로 다른 시스템을 상호 연동해 동작시키기 위한 표준화라는 연동 형식의 통일필요
- ex) 프린터 - A4용지
##### 프로토콜
- 상호 연동되는 시스템이 데이터를 교환할때 표준화된 특정 규칙인 프로토콜을 따름
- 정보의 형식, 절차적 순서 등에 초점
- 시스템과 동등관계
##### 인터넷 
- 전 세계의 모든 네트워크가 유기적으로 연결되어 동작하는 통합 네트워크
- 데이터 전달기능에선 공통으로 IP프로토콜을 사용
- 인터넷은 IP(Internet Protocol)의 첫단어에서 유래
#### 시스템 구분
##### 노드
- 인터넷에 연결된 시스템을 가장 일반화한 용어
- 데이터를 주고 받을 수 있는 모든 시스템을 통칭하여 사용
##### 호스트
- 컴퓨팅 기능이 있는 시스템
##### 클라이언트
- 호스트 사이에 제공되는 서비스를 기준으로 클라이언트와 서버로 나누지만 고정은 아님, 클라이언트가 서버가 될수도 있음
- 서비스를 이용하는 시스템
##### 서버
- 서비스를 주고받는 호스트들의 관계에서 임의의 서비스를 제공하는 시스템
- 클라이언트보다 먼저 실행되어 요청을 대기 
##### FTP(File Transfer Protocol)
- 파일 송수신 기능
##### 텔넷(Telnet)
- 네트워크를 이용해 원격지의 서버에 접속하여 데이터를 교환할 수 있는 프로토콜
- 통신과 네트워크 단어의 축소한 용어
- 원격 단말기 기능


##### 진법(notation)
- 디지털 컴퓨터는 2진수 기반으로 숫자를 표현하기 위해 많은 자릿수를 차지하고, 10진수에 비해 사람이 인식하기 어렵기 때문에 2의 승수이면서 10진수와 가까운 8진수와 16진수를 컴퓨터에서 많이 사용
- 2진수: 2개(0, 1)의 숫자로 표현
- 8진수: 8개(0~7)의 숫자로 표현
- 10진수: 10개(0~9)의 숫자로 표현
- 16진수: 16개(0~9, A, B, C, D, E, F)의 숫자로 표현
자릿수 ex) 259 = 2x10^2 + 5x10^1 + 9x10^0

##### 진법 변환
30(8진수) = 3x8^1 + 0x8^0 = 24(10진수)
54(16진수) = 5x16^1 + 4x16^0 = 84(10진수)

##### BCD코드(Binary coded decimal)
- 십진수 한 자리를 4비트의 2진수로 표기
- 9(1001)가 최대
- ex) (0001 0000)BCD = (10)10

##### BCD덧셈
- (1010 ~1111) 6개의 숫자가 남기 때문에 2진수 덧셈에 보정이 필요
- 2진수 합이 1010보다 크거나 같으면 0110(6)을 더해줌
- ex) 4+5 = 9 -> 0100 + 0101 =1001
-  4+8 = 12 -> 0100 + 1000 = 1100 + 0110(6) = (1 0010)BCD
##### 비트 표현 개수
- 4비트로 나타낼 수 있는 최대숫자 개수 2^4 = 16 (0~15)
- 8비트로 나타낼 수 있는 최대숫자 개수 2^8 = 256 (0~255)
- 16비트로 나타낼 수 있는 최대숫자 개수 2^16 = 65536 (0~65535)
- 1바이트 = 256개의 숫자 표현 (0~255)
- nible = 4 bit 
- byte = 8 bit 
- word = 2 byte
- quad word = 4 word = 8 byte
- octo word = 8 word = 16 byte

##### 보수(Complement)
- 컴퓨터는 2진수로 표현되기 때문에 음수를 보수를 이용해 표현
- 보수는 보충을 해주는 수
ex) 1에 대한 10의 보수는 9
4에 대한 16의 보수는 12

#### 보수를 이용한 음의 정수 표현
##### 부호 절대값
- 최상의 비트(MSB(Most Significant Bit))로 음수 표현
- 0이 양수와 음수로 나누어지고 덧셈, 뺄셈이 매우 불편(큰수가 위로오게하고 부호 설정)
- -(2^(n-1)-1)~(2^(n-1)-1) , 8비트 -127 ~ 127
ex) 0000 000 = +0
1000 0000 = -0
##### 1의 보수
- 2진수를 각 자리수의 값이 1인 수를 빼면 1의 보수
- 0이 양수와 음수로 존재
- -(2^(n-1)-1)~(2^(n-1)-1) \, 8비트 -127 ~ 127
ex) 11111111 - 00000011(3) = 11111100(-3)
##### 1의 보수 뺄셈(추가)
- 빼는 수대신 빼는 수의 1의 보수를 더한다음 자리올림이 생기면 최상의 비트를 제외하고 최하위 비트에 +1, 자리올림이 없으면 1의 보수를 구한 후 -부호를 붙임
ex) 1101 -1001 = 1101+0110 = 10011 -> 0100
0101 -1001 = 0101 + 0110 = 1011 -> -0100
##### 2의 보수
- 1의 보수에서 1을 더한 값
- 0 음수 존재x
- 컴퓨터에서 사용
-  -2^(n-1) ~ (2^(n-1) - 1) ,  8비트 -128 ~ 127
ex) 11111111 - 00000011(3) = 11111100 +1 = 111111101(-3)
##### 2의 보수 뺄셈(추가)
- 빼는 수의 2의 보수를 구하고 더한다음 자리올림이 생기면 자리올림을 제외한 나머지 부분, 생기지않으면 연산결과의 2의보수를 구한 후 -부호

#### 컴퓨터를 구성하는 3대 요소
- CPU(ALU(Arithmetic Logic Unit)+CU(Control Unit)), Memory, IO(입출력) 장치
##### 1) 중앙처리장치(CPU, Central Processing unit)
- 메모리에 저장된 명령어를 처리한다.
###### (1) ALU(Arithmetic Logic Unit)
- 데이터 처리
- 산술/논리 연산을 수행하는 장치
###### (2) CU(Control Unit)
- 명령어 레지스터에 저장된 명령어를 해석한다.
- 컴퓨터 구성요소를 제어할 제어신호 생성한다.
###### (3) Register
- CPU 내부에(CPU와 직접 연결된) Register가 존재해, Register를 이용하면 일반메모리보다 빠른 연산속도로 처리 가능하다.
###### (4) 내부 버스
- CPU 내부의 구성요소 간 데이터 전달과 연결을 위한 경로이다.
##### 2) Memory
- 데이터나 프로그램(명령어)을 저장하는 곳 이다.
- 주기억장치에는 램(RAM), 롬(ROM) 등이 있다.
##### 3) 입출력 장치
- 사용자 또는 컴퓨터 외부에서 데이터를 입력받고 출력하기 위한 장치이다.
- 모니터, 키보드, 마우스 등이 있다.
#### 물리 계층의 유무선 전송매체 요약
- 전송매체는 송신기와 수신기 사이의 물리적인 전송 통로이며, 유선 매체와 무선 매체로 나눌 수 있다.
- 전송매체는 개별 정보의 비트 교환(Bit) 문제를 다룬다.
- 유선 매체로서는 트위스트 페어 케이블(랜 구성에 사용, Shield의 유무로 UTP케이블, STP케이블로 분류), 동축 케이블(단일 전선과 도체로 구성), 광섬유 케이블(코어, 클래딩, 코팅으로 구성) 등이 있다. 주로 통신 경로의 길이, 전송속도에 따라 선택함
- 무선 매체로서는 크게 라디오파(3KHz~1GHz), 마이크로파(1GHz~300GHz), 적외선(300GHz~400THz)으로 분류한다.
- 물리 계층에서 다루는 전송매체의 특성에는 전송 속도, 송수신 호스트 사이의 클록 동기화 방법, 물리적 연결 형태 등이 있다.

#### 표준화 기구 ISO, IEEE, ITU, IETF, ANSI, UnicoeConsortium, TTA
##### 국제 표준화 기구 (ISO : International Organization for Standardization )
-  스위스 제네바에 본부를 둔 비정부 기구 민간 기구 ISO는 여러 나라의 표준 제정 단체들의 대표들로 이루어진 국제적인 표준화 기구이다. 
- ISO는 1947년에 출범되어 각 나라마다 다른 산업, 통상 표준의 문제점을 해결하고자 국제적으로 통용되는 표준을 개발하고 보급한다. 
- ISO가 정한 표준은 보통 국제 협약이나 국가 표준 제정 시 광범위하게 인용, 활용되기 때문에 국제적인 영향력이 크다.
##### 전기 전자 기술자 협회(IEEE : Institute of Electrical and Electronics Engineers )
- 미국 뉴옥에 위치한 IEEE는 전기전자공학 전문가들의 국제조직이다.
- IEEE는 전기전자공학에 관한 최대 기술 조직으로 주요 표준 및 연구 정책을 발전시키고 있다.
- IEEE는 전기 전자에 관한 산업 표준 회의를 통하여, 산업 기기간의 표준화를 구현한다.

##### 국제전기통신연합(ITU: International Telecommunication Union )
- 스위스 제네바에 본부를 두고 있는 ITU는 전기 통신의 개선과 효율적인 사용과 회원국간 조화로운 전기통신 수단 사용 보장을 목적으로 하는 정부간 국제 기구이다.
- 정보통신기술 관련 문제를 책임지는 유엔 전문 기구이며 유엔 개발 그룹에 속해 있다.
- 현존하는 국제 기구 중에서 가장 오랜 역사를 가졌다.
##### 국제 인터넷 표준화 기구(IETF: Internet Engineering Task Force )
- IETF는 인터넷의 표준화를 다루기 위해, 1986년 인터넷 아키텍처 위원회(IAB)의 산하기구로 설립됩 인터넷 표준화 작업기구 이다.
- IETF는 인터넷의 운영, 관리 및 기술적인 쟁점 등을 해결하는 것을 목적으로 망 설계자, 관리자, 연구자, 망 사업자 등으로 구성된 개방된 공동체이다
##### 미국 국립표준 협회(ANSI: American National Standards Institute )
- ANSI는 미국에서 제품, 서비스, 과정, 시스템, 인력관리 분야에서 산업 표준을 제정하는 비영리 민간 기구이다.
- 또한 ISO에 가입되어있고, ANSI는 미국의 제품이 전세계에서 사용될 수 있도록 미국의 표준을 국제표준에 맞추는 활동도 한다.
##### 유니코드 컨소시엄(Unicode Consortium)
- 미국 캘리포니아에 위치한 Unicode Consortium은 크기와 범위 제한이 있으면서 다언어 환경과 호환되지 않는 기존 문자 인코딩 스킴을 대체할 목적으로 개발된 유니코드 표준을 관리하고 출판하는 비영리 단체이다.
##### 한국정보통신기술협회(TTA: Telecommunications Technology Association)
- 대한민국 경기도에 위치한 TTA는 대한민국의 기술 연구 단체로 정보통신기술(ICT)의 표준 제정, 보급 및 표준제품의 시험인증을 위해 설립된 법정 기관이다.
- 1988년 설립된 TTA는 국내 유일 정보통신 단체표준 제정기관이며, 30년 넘는 기간 동안 대한민국 ICT 표준화와 시험인증의 구심점 역할을 해 왔을 뿐만 아니라, 국제 표준화를 선도함으로써 국제사회에서 대한민국의 ICT 산업 성장에 크게 기여했다.


#### Protocol. Interface, standardization(표준화)설명
##### 프로토콜(Protocol)
- 상호 연동되는 시스템이 데이터를 교환할때 표준화된 특정 규칙을 따르는데 이를 프로토콜이라 한다.
- 프로토콜은 주고받는 정보의 형식, 그 과정에서 발생하는 절차적 순서에 무게를 둔다.
- 프로토콜은 동등한 위치에 있는 시스템 사이의 규칙으로 인터페이스와 구분된다.
##### 인터페이스(Interface)
- 시스템끼리 연결하기 위한 표준화된 접근 방법이다.
- 두 시스템이 연동하기 위한 특정한 접촉 지점(access point)을 의미한다.
- 시스템과 전송매체의 연결 지점에 대한 규격을 나타낸다.
- 연결되는 시스템끼리 동등관계 또는 상하관계이다.
##### 표준화(standardization)
- 사물, 개념, 방법 및 절차 등에 대하여 합리적인 표준(Standard)을 설정하고, 이를 따르고 활용하기 위한 규칙, 지침, 가이드 등을 만드는 조직적 행위를 말한다.
- 프로토콜처럼 서로 다른 시스템을 상호 연동해 동작시키기 위한 표준화라는 연동 형식의 통일필요
- ex) 프린터 – A4용지(표준화된 용지)
#### Client-Server Model, Peer-to-peer Model설명
##### 클라이언트 서버 모델(Client–Server Model)
- 서비스 요청자인 클라이언트와 서비스 자원의 제공자인 서버 간에 작업을 분리해주는 분산 애플리케이션 구조이자 네트워크 아키텍처를 나타낸다.
- 클라이언트는 서비스를 이용하는 시스템이고, 서버는 서비스를 제공하는 시스템이다.
- 호스트는 다양한 서비스를 주고받기 때문에, 임의의 호스트가 클라이언트나 서버로 고정되지 않는다.
- 일반적으로 서버는 클라이언트보다 먼저 실행 상태가 되어 클라이언트의 요청에 대기 해야하며, 지속적으로 클라이언트의 요청에 서비스를 반복 제공해야 한다.
###### 장단점
- 서버가 집중적으로 서비스 관리를 하기 때문에 시스템의 설계유지가 편하다.
- 서버에 문제가 생기면 전체가 중단된다.
###### 동등 계층간 통신망(Peer-to-peer Model)
- 비교적 소수의 서버에 집중하기보다는 망구성에 참여하는 기계들의 계산과 대역폭 성능에 의존하여 구성되는 통신망이다.
- 네트워크에 연결된 모든 컴퓨터들이 서로 대등한 동료의 입장에서 데이터나 주변장치 등을 공유할 수 있다.
- 프로토콜처럼 동등한 위치에 있다.
###### 장단점
- 높은 확장성을 가진다.
- 네트워크 전송시간으로 인해 노드간 정보 불일치, 성능 저하가 발생한다.

#### OSI 7 계층
###### 물리 계층(Physical Layer)
- 물리계층은 물리적으로 데이터를 전송하는 역할을 수행한다.
- 호스트를 전송매체와 연결하기 위한 인터페이스 규칙과 전송매체의 특성을 다룬다.
- 이 전송 매체는 비트(bit) 교환 문제를 다룬다.
- 물리 계층은 하드웨어 시스템으로 구현된다.
- 물리 계층에서 다루는 전송 매체의 특성에는 데이터의 전송 속도, 송수신 호스트 사이의 클록 동기화 방법, 물리적 연결 형태 등이 있다.
- 이 계층에 속하는 대표적인 장치는 통신 케이블, 허브, 리피터 등이 있다.
- 전송 데이터 단위는 bit이다.

###### 데이터 링크 계층(Data Link Layer)
- 물리 계층을 통해 데이터를 전송하는 과정에서 잡음(Noise)등과 같은 물리적 전송 오류를 해결한다.
- 물리적인 전송 오류를 감지(Sense)하고 일반적인 컴퓨터 네트워크에서 오류제어(Error Control)는 송신자가 원래의 데이터를 재전송(Retransmission)하는 방식이다.
- Frame 헤더에 표시되는 송수신 호스트 정보에 LAN카드에 내장된 송수신 호스트의 MAC주소가 기록된다.
- 물리 계층을 통해서 데이터를 전송할 때 이 MAC주소를 이용해서 호스트를 구분한다.
- 흐름 제어 기능도 지원한다.
- 이 계층에 속하는 대표적인 장치는 스위치와 브릿지가 있다.
- 전송 데이터 단위는 DPDU(Data Link Protocol Data Unit)이며 Frame이라 부른다.
###### 네트워크 계층(Network Layer)
- 데이터가 여러 개의 중개 시스템을 거칠 때, 데이터가 올바른 경로를 선택할 수 있도록 지원하는 계층이다. 즉 라우팅 문제를 처리한다고 말한다.
- 중개 시스템의 기능은 일반적으로 라우터(Router)가 수행한다.
- 전달 경로 선택은 미리 정해지는 정적인(Static) 방식과 현재 부하 상태에 따라 결정되는 동적인(Dynamic) 방식으로 구분한다.
- 네트워크 부하가 증가하면(패킷 증가) 특정 지역에 혼잡(Congestion)이 발생할 수 있는데, 혼잡제어도 네트워크 계층이 담당한다.
- 중개 과정에서 경로선택의 기준이되는 호스트 주소가 필요하다.
- 인터넷에서 IP 프로토콜이 네트워크 계층의 기능을 수행하며, 호스트의 IP주소가 경로선택에 중요한 기준이 된다.
- 이 계층에 속하는 대표적인 장치로 라우터(Router), Layer, 스위치가 있다.
- 전송 데이터 단위는 NPDU(Network Protocol Data Unit))이며 패킷(packet)이라 부른다.
- TCP/IPProtocolSuite로는 IP, ICMP, IPsec, ARP, RIP, BGP가 존재한다.
###### 전송 계층(Transport Layer)
- 송신 프로세스와 수신 프로세스를 직접 연결하는 단대단(End-to-End) 통신 기능을 제공한다.
- 송신 프로세스와 수신 프로세스 간의 연결 기능을 제공하기 때문에, 프로세스 사이의 안전한 데이터 전송을 지원한다.
- 전송계층은 데이터가 전송되는 최종적인 경로상의 양 끝단 사이의 연결이 완성되는 곳이다.
- 인터넷에서 TCP, UDP가 전송계층의 기능을 수행한다.
- 대표적인 장치로는 게이트웨이가 있다.
- 전송 데이터 단위는 TPDU(Transport Protocol Data Unit)이며 TCP에서는 세그먼트(Segement)라고 부르고 UDP에서는 데이터그램(Datagram)이라고 부른다.
- TCP/IPProtocolSuite로는 TCP, UDP, RTP, SCTP가 존재한다.
###### 세션 계층(Session Layer) 
- 전송 계층에서 제공하는 연결의 개념과 유사한 세션 연결을 지원하지만, 사용자에게 원격 파일 전송이나 원격 로그인 등과 같은 상위의 논리적 연결개념인 세션 기능을 제공한다.
- 송수신 호스트 사이의 대화 제어와 상호 배타적인 동작을 제어하기 위한 토큰 제어, 일시적인 전송 장애를 해결하기 위한 동기 기능(Synchronizaion)을 제공한다.
- 대표적으로 RPC, Socket 등이 있다.
- 전송 데이터 단위는 SPDU(Session Protocol Data Unit)이다.
- TCP/IPProtocolSuite로는 TCP의 세션 관리 부분이 있다.
###### 표현 계층(Presentation Layer)
- 표현 계층은 전송되는 데이터의 의미를 잃지 않도록 올바르게 표현하는 방법을 다룬다.(데이터 코딩)
- 즉, 정보를 교환하는 시스템이 표준화된 방법으로 데이터를 인식할 수 있게 해준다.
- 압축으로 전송 데이터의 양을 줄이고, 암호화로 데이터를 안전하게 보호하는 기능도 다루고 있다.
- 전송 데이터 단위는 PPDU(Presentation Protocol Data Unit)이다.
- TCP/IPProtocolSuite로는 XDR이 존재한다.
###### 응용 계층(Application Layer)
- 응용 계층은 사용자를 위한 다양한 네트워크 응용 환경을 지원한다.
- 응용 환경은 매우 다양하지만 인터넷에서 많이 사용하는 서비스 FTP(File Transfer Protocol)로부터 시작된 파일공유서비스이다.
- 대표적으로 우리가 잘 알고있는 HTTP, FTP 등의 프로토콜이 응용 계층에 속한다.
- 응용 서비스 예로 텔넷(Telnet)이 제공하는 가상터미널, 전자메일이 있다.
- 전송 데이터 단위는 APDU(Application Protocol Data Unit)이다.
- TCP/IPProtocolSuite로는 HTTP, SMTP, SNMP, FTP, 텔넷, NFS, NTP가 존재한다.
###### TCP/IPProtocolSuite
- 인터넷에서 컴퓨터들이 서로 정보를 주고받는 데 쓰이는 통신규약(프로토콜)의 모음이다. 인터넷 프로토콜 슈트 중 TCP와 IP가 가장 많이 쓰이기 때문에 TCP/IP 프로토콜 슈트라고도 불린다. 
- 인터넷에 연결하고자 하는 호스트는 반드시 네트워크 계층의 IP프로토콜을 지원해야 하며, 전송 계층은 TCP나 UDP를 사용한다.
- 현재 인터넷이 주로 사용하는 IP 프로토콜은 버전 4(IPv4) 이다.
###### 네트워크 인터페이스 계층 (Network Interface Layer)
- 이 계층은 Node-To-Node간의 신뢰성 있는 데이터 전송을 담당하는 계층이다.
- OSI7 계층의 물리 계층과 데이터링크 계층의 역할을 바로 이 계층이 담당한다.
- 주요 프로토콜로 LAN상에서는 Ethernet, TokenRing, FDDI 등이 있으며 WAN 상에서는 X.25, Frame Relay, PPP 등이 있다.
###### 인터넷 계층 (Internet Layer)
- OSI7계층의 네트워크 계층을 담당하는 계층이다.
- OSI7 계층처럼 호스트간의 라우팅을 담당한다.
- 주요 프로토콜로  IP, ARP, RARP, ICMP, IGMP등이 있다.
###### 전송 계층 (Transport Layer)
- OSI7 계층의 전송계층과 같다.
- 프로세스간의 신뢰성 있는 데이터 전송을 담당하는 계층이다.
- 주요 프로토콜로 TCP, UDP등이 있다.
###### 응용 계층 (Application Layer)
- 사용자와 가장 가까운 계층이다.
- OSI7계층의 5계층부터 7계층까지의 기능을 담당한다.
- 응용 프로그램이 이 계층에서 동작한다. 
- 주요 프로토콜로 HTTP, SMTP, SNMP, FTP, 텔넷, NFS, NTP이 존재한다.
##### IPv4와 IPv6
- IPv4는 32비트의 주소공간을 제공함에 반해, IPv6는 128비트의 주소공간을 제공한다.
- IPv4의 최대 주소 공간의 크기가 32비트로 제한되면서 확장성에 많은 문제점이 야기되었다. 특히 인터넷 요구 사항의 폭증을 수용하기 위해서는 IPv6로의 전환이 필수적인데 IPv6은 IPv4의 주소가 바닥나는 것을 막는 것 외에도 패킷 크기 확장, 효율적인 라우팅 등에 장점이 있다.

##### 인터넷프로토콜(IP) 주소체
- 송신 호스트와 수신 호스트가 패킷 교환 네트워크에서 정보를 주고받는 데 사용하는 정보 위주의 규약(프로토콜, Protocol)이다,
- 인터넷에서 네트워크 계층에서 기능을 수행하는 IP가 호스트를 구분하기 위하여 사용하는 주소 체계이다.
- 즉, 인터넷에서 IP주소는 패킷의 경로를 결정하는데 중요한 역할을 한다.
- IP 주소는 일반적으로 32비트의 이진 숫자로 구성되며, 보통 8비트씩 4부분으로 나누어 십진수로 변환한 후 점으로 구분하여 표기한다. (ex 255.255.255.255 8비트로 표현할 수 있는 최대 숫자가 255이므로 범위는 0~255이다.)
- IP주소는 유일성을 보장하기 위해 국제 표준화 기구가 전체 주소를 관리하고 할당함으로써, 중복 주소의 사용을 원천적으로 차단한다.
- 보편적으로 사용되는 IP주소 체계는 IPv4(32비트)와 IPv6(128비트)이다.
- 현재의 IP 프로토콜은 IPv6와 구분짓기 위해 IPv4로 표현한다.

##### HTTP(HyperText Transfer Protocol)
- HTTP는 웹 상에서 정보를 주고받을 수 있는 프로토콜이다. 
- 주로 HTML 문서를 주고받는데 쓰인다.
- HTTP는 연결 상태를 유지하지 않는 비연결성 프로토콜이다.
- 따라서 HTTP는 클라이언트와 서버 사이에서 발생하는 request & response 방식으로 동작한다.
- TCP/ IP를 이용하는 응용 프로토콜이다.
##### DNS(Domain Name System)
- 초기 인터넷에서는 호스트 파일에 수동적으로 호스트이름과 IP주소의 조합을 기록, 관리하였지만, 인터넷이 발달함에 따라 갱신하고 호스트가 증가하면서 갱신하고 복사하는 작업에 많은 시간과 노력이 필요했다.
- 이런 문제점을 해결하기 위해 DNS라는 주소와 이름 정보를 자동으로 유지하고 관리하는 분산 데이터 베이스 시스템이 등장하였다.
- 이 DNS는 1983년 Jon Postel과 Paul Mockapetris로 인해 처음 등장하였다.
- DNS는 호스트의 도메인 이름을 호스트의 네트워크 주소로 바꾸거나 그 반대의 변환을 수행할 수 있도록 하기 위해 개발되었다. 
- 즉, 특정 컴퓨터(또는 네트워크로 연결된 임의의 장치)의 주소를 찾기 위해, 사람이 이해하기 쉬운 도메인 이름을 숫자로 된 식별 번호(IP 주소)로 변환해 준다. 
- DNS 시스템은 호스트 이름을 국가 도메인, 단체 종류, 단체 이름, 호스트라는 네 계층 구조로 나누고 점으로 구분해서 표기한다. (호스트.단체이름.단체종류.국가도메인)
- DNS는 호스트의 정보를 여러 네임 서버에 분산하여 관리하도록 설계되었고, 계층 구조로 연결된 네임 서버는 자신의 영역에 해당하는 호스트 정보만 관리하며, 정보를 상호 교환하는 협력관계를 통해 전체 호스트 정보를 일관성 있게 유지한다.
##### 오류 제어(Error Control)
- 호스트 간의 데이터 전송이 일어날 때 데이터의 오류에 관련된 제어를 하는 역할을 한다.
- 전송 오류에는 데이터 변형오류와 데이터 분실오류가 있다.
- 전송 오류를 해결하는 오류 제어(Error control)는 통신 프로토콜의 가장 기본적인 기능에 속하며, 일반적으로 컴퓨터 네트워크에서 송신자가 원래의 데이터를 재전송하는 방식으로 처리한다.
- 1차적으로 물리계층의 전송매체에 의한 물리적인 오류는 데이터 링크 계층에서 해결하고 상위 계층의 프로토콜이 수핼될 때 발생되는 논리적인 전송 오류는 상위 계층에서 다룬다.
- 보통 오류감지는 송신호스트에서 이루어지며, 오류 검출은 수신 호스트에서 이루어지고, 오류제어는 송신 호스트에서 재전송으로 이루어진다.
- 자동반복 요청 (ARQ: Automatic Repeat Request) 기법을 사용해 프레임이 손상되었거나 손실되었을 경우, 재전송을 통해 오류를 복구한다.
1) Stop and Wait ARQ
- 전송측은 수신측에서 보내준 ACK를 받을 때까지 프레임의 복사본을 유지한다.
- 만약 데이터나 ACK가 분실되었을 경우 일정 간격의 시간을 두고 타임아웃이 되면 송신측은 데이터를 재전송한다.
2) Go-Back-n ARQ(GBn ARQ)
- 전송된 프레임이 손상되거나 분실될 경우, 확인된 마지막 프레임 이후로 모두 재전송하는 기법이다.
##### 흐름 제어(Flow Control)
- 송신 측과 수신 측이 서로 데이터를 주고 받을 때, 여러가지 요인에 따라 처리 속도가 달라질 수 있다.
- 일반적으로 수신 호스트의 버퍼 처리 속도보다 송신 호스트의 데이터 전송 속도가 빠를 경우 논리적인 데이터 분실 오류가 발생할 수 있다.
- 따라서, 흐름제어는 이러한 송신측과 수신측 간에 데이터 처리 속도의 차이(흐름)로 인해 발생하는 문제를 해결하기 위한 방법이다.
1) 정지- 대기(Stop and Wait)
- 매번 전송한 패킷에 대한 확인 응답을 받아야 그 다음 패킷을 전송할 수 있다. 
- 구조가 간단한 대신, 하나를 주고 하나를 받기 때문에 비효율적이다.
2) 슬라이딩 윈도우(Sliding Window)
- 수신측에서 설정한 윈도우 크기만큼 송신측에서 확인 응답 없이 세그먼트를 전송할 수 있게 하여 데이터 흐름을 동적으로 조절하는 기법이다. 
##### 3가지 데이터 전달 모드
1. 단방향(Simplex) 방식
- 데이터를 한쪽 방향으로만 전송하는 것, 즉 데이터 송신만 가능 또는 수신만 가능 (ex 방송, 감시 카메라)
2. 반이중(Half Duplex) 방식
- 데이터를 양방향으로 전송이 가능하지만, 특정 시점에는 한쪽 방향으로 전송하는 것, 즉 데이터 송수신은 가능하지만, 동시에 불가(ex. 무전기)
3. 전이중(Full Duplex) 방식
- 데이터를 양방향으로 동시에 전송하는 것, 즉 동시에 데이터 송수신 가능(ex 전화기)
##### 분할 및 재조립(Segmentation and Reassembly)
- 분할 및 재조립은 패킷교환방식의 통신망에서 패킷을 전송 전에 작은 단위로 나눈 후, 수신 측에서 올바른 순서대로 패킷을 재조립하는 과정을 말한다.
- 패킷을 작게 만드는 이유는 네트워크를 통해 패킷을 고속으로 보내기 위함이고 정보 전송시 오류를 줄이거나 또는 주어진 경로에서 허용하는 패킷 크기의 제한 때문이다. 
- 분할 및 재조립은 통신 양단의 전송 계층에서 수행된다. 송신측이 데이터를 분할(Segmentation)하여 수신측이 정보를 재조립(Reassembly)한다.
##### Connection-oriented와Connectionless Service. TCP와 UDP
###### Connection Oriented Service(연결지향형)
- 연결형 서비스를 이용하는 절차는 크게 3가지 이다.
- 송신자와 수신자 사이에 먼저 (1)데이터 전달 경로를 설정하는 연결이 설정이된 후 (2)데이터가 전송된다.
- 모둔 데이터 전송이 완료된 후 연결을 끊는 (3)연결 해제 단계가 있다.(ex 통화)
- TCP는 Connection-Oriented 프로토콜이며, 1:1로 연결 상태를 유지하여 통신하는 것을 말한다.
- 프로토콜에 의해 연속적으로 패킷의 상태 정보 유지
###### Connectionless Service(비 연결지향형)
- 데이터는 연결 설정이나 연결 해체의 필요 없이 한쪽에서 다른 쪽으로 보내진다.
- 즉 데이터는 독립적인 경로 선택 과정에 의해 전달되므로 도착하는 순서가 보낸 순서와 일치하지 않을 수 있다. (ex 우편)
- IP와 UDP 는 Connectionless 프로토콜이며, 연결 상태를 유지하는 것이 아니라 주소를 가지고 다음 라우터나 목적지로 전달만하여 데이타그램에 대해 관여 하지 않는다.
###### TCP(Transmission Control Protocol)
- 연결형 서비스를 지원하는 전송 계층 프로토콜로서, 인터넷 환경에서 기본으로 사용한다.
- TCP/UDP는 시스템 운영체제인 커널 내부에 구현된다.
- 따라서 전송 계층의 기능을 제공하는 소켓 시스템 콜(운영체제)을 호출해 TCP/UDP 기능을 사용할 수 있다.
- 혼합제어와 흐름제어를 지원한다.
- 바이트 스트림을 통한 연결을 한다.
- 순서가 보장되고, 상대적으로 느리다.
- 신뢰성 있는 데이터를 전송한다.(reliable)
- 데이터 단위: Segment
- connection-oriented
- HTTP, Email, File transfer에서 사용한다.
###### UDP(User Datagram Protocol)
- 비연결형 서비스를 지원하는 전송계층 프로토콜로서, 인터넷상에서 서로 정보를 주고받을 때 정보를 보낸다는 신호나 받는다는 신호 절차를 거치지 않고, 보내는 쪽에서 일방적으로 데이터를 전달하는 통신 프로토콜이다.
- TCP/UDP는 시스템 운영체제인 커널 내부에 구현된다.
- 따라서 전송 계층의 기능을 제공하는 소켓 시스템 콜(운영체제)을 호출해 TCP/UDP 기능을 사용할 수 있다.
- 혼합제어와 흐름제어를 지원하지 않는다.
- 메세지 스트림을 통한 연결을 한다.
- 순서가 보장되지 않고, 상대적으로 빠르다.
- 신뢰성 없는 데이터를 전송한다.(unreliable)
- 데이터 단위: Datagram
- connectionless
- DNS, Broadcasting에서 사용한다.
##### ARP, RARP, BOOTP, ICMP, DHCP
###### ARP (Address Resolution Protocol)
- 인터넷 통신을 하기 위해 송수신의 IP주소와 MAC주소가 필요한데, 사용자로부터 입력된 수신자의 IP 주소를 이용해 MAC주소를 구하는 기능이다.
- 즉 ARP는 원격 호스트의 주소 변환 기능을 제공한다.
###### RARP(Revers Address Resolution Protocol)
- LAN 카드에 보관된 MAC주소를 이용해 IP주소를 제공하는 프로토콜이다.
- 일반 컴퓨터 시스템은 로컬 호스트의 IP주소가 파일 시스템에 보관되므로 RARP를 사용하지 않지만, 디스크가 장착되지 않은 시스템에서는 RARP를 반드시 사용해야한다.
- 즉, RARP는 로컬 호스트의 주소 변환 기능을 제공하다.
###### ICMP(Internet Control Message Protocol)
- IP동작 과정에서 데이터 전송 오류가 발생하면 오류 메시지(Requested service is not available 등)를 전송하는 기능을 담당하는 프로토콜이다.
- IP와 같은 네트워크 계층 프로토콜로 간주할 수 있지만, ICMP에서 발생하는 ICMP 메시지는 IP 패킷에 캡슐화되어 전송된다.
###### BOOTP(Bootstrap Protocol)
- 디스크가 없는 호스트에 대해 네트워크 장치에 IP 주소 및 구성 정보를 할당하기 위해 인터넷 프로토콜 네트워크에서 사용되는 컴퓨터 네트워킹 프로토콜이다.
- 클라이언트/서버 구조로 동작하는 정적인 주소 설정 방식이다.
- UDP에 의해 캡슐화되고, 보통 TFTP와 함께 동작한다.
- DHCP와 완벽호환 되는 특징이 있어서, BOOTP 클라이언트가 DHCP 서버와 통신이 가능하다.
###### DHCP(Dynamic Host Configuration Protocol)
- DHCP는 네트워크에 접속한 장치의 ip, subnet mask, gateway address, DNS와 같은 정보를 자동으로 설정해주는 기술
- BOOTP에서 발전된 동적 주소 할당 프로토콜이다.
- 이용자가 네트워크 정보를 직접 설정할 필요 없이 자동으로 그 설정이 가능하기 때문에 네트워크 관리의 용이성을 제공해준다.
##### Congestion Control
- 컴퓨터 네트워크를 이용해 전송되는 패킷이 지나치게 많아지면 네트워크의 전송속도가 감소하게 되어, 프로토콜 동작에 많은 영향을 끼치게 되는데, 이러한 네트워크의 트래픽이 과도하게 증가하는 문제를 조절하는 것이 혼잡 제어(Congestion Control)이다.
- 흐름제어가 송신측과 수신측 사이의 전송속도를 다루는데 비해 혼잡제어는 호스트와 라우터를 포함한 보다 넓은 관점에서 전송 문제를 다룬다.
- 네트워크 계층(Network Layer)에서 담당한다.
- 혼잡 제어는 일반적으로 4가지 방식이 있다.
###### 합 증가/곱 감소(AIMD:Additive Increase / Multiplicative Decrease)
- 처음에 패킷을 하나씩 보내고, 문제없이 도착하면 단위 시간 내에 보내는 패킷의 수(window 크기)를 1씩 증가(합 증가)시켜가면서 전송하는 방법이다. 패킷 전송이 실패하거나 일정한 시간을 넘으면 패킷을 보내는 속도를 절반으로 줄이게 되는 방식이다.(곱 감소)
- AIMD 방식이 네트워크의 수용량 주변에서는 효율적으로 작동하지만, 처음에 전송속도를 올리는데 시간이 오래 걸리는 단점이 있다.
###### 느린 시작(Slow Start)
- 느린 시작 방식은 AIMD방식처럼 패킷을 하나씩 보내어 패킷이 문제없이 도착하면 각각의 응답문자(ACK: Acknowledgement code) 패킷마다 단위 시간 내에 보내는 패킷의 수(window 크기)를 1씩 늘린다.
- 한 주기가 지나면 window 크기는 2배가 되고 전송속도는 지수 함수형태로 증가하게 된다.
- 혼잡 현상이 발생하면 window 크기는 1로 떨어뜨린다.
###### 빠른 재전송(Fast Retransmit)
- 빠른 재전송은 TCP의 혼잡 제어에 추가된 정책이다.
- 패킷을 받는 쪽에서 먼저 도착해야 할 패킷이 도착하지 않고 다음 패킷이 도착한 경우에도 ACK 패킷을 보내고, 순서대로 잘 도착한 마지막 패킷의 다음 패킷의 순번을 ACK 패킷에 실어서 보낸다.
- 중간에 패킷 하나가 손실되게 되면 보내는 측에서는 순번이 중복된 ACK 패킷을 받게 되고, 중복된 순번의 패킷을 3번 받으면 재전송을 한다. 이런 혼잡으로 인해 window 크기를 줄인다.
###### 빠른 회복(Fast Recovery)
- 혼잡한 상태가 되면 window 크기를 1로 줄이지 않고 반으로 줄이고 선형 증가시키는 방법이다. 
- 혼잡 상황이 한번 발생한 후, 순수한 AIMD 방식으로 동작하게 된다.
##### End-to-end 전송계층
- 송신 프로세스와 수신 프로세스를 직접 연결하는 단대단통신(End-to-End)을 하는 OSI 계층은 전달(송)계층(Transport Layer)이다.
##### 회선 교환(Circuit Switching)
- 연결형 서비스(Connetion-oriented)
- 데이터 전송 전 고정된 연결 경로 미리 설정
- 고정된 대역폭의 전송률로 모든 데이터가 같은경로로 전달
- 안정적인 데이터 전송률 지원, 라우팅 작업이 상대적으로 쉬움
##### 메시지 교환(Message Switching)
- 경로 미리 설정하지 않고, 전송하는 메시지의 헤더마다 목적지 주소를 표시
- 가변 대역폭
- 중간의 교환 시스템은 전체 메시지가 도착할 때까지 받은 메시지를 버퍼에 일시적으로 저장하여 모든 메시지가 도착하면 하나의 단위로 다음 교환 시스템에 전달하는 방식으로 데이터 전송이 교환기 단위로 이어짐, 즉 전체 데이터가 하나의 단위로 교환 처리 
- 교환 시스템에서 전송 데이터를 저장하는 기능을 제공하여, 송신 호스트가 보낸 시점과 수신 호스트가 받은 시점이 반드시 일치할 필요가 없음
##### 패킷 교환(Packet switching)
- 비연결형 서비스(Connectionless)
- 데이터 전송 전 경로 미리 설정하지 않음
- 가변 대역폭
- 데이터를 패킷이라는 일정크기로 나누어 전송하며, 각 패킷은 독립적으로 라우팅 과정을 거쳐 목적지에 도착
- 전송 대역의 효율적 이용, 호스트의 무제한 수용, 패킷에 우선순위 부여

##### 가상 회선 방식(Virtual Circuit)
- 가상 회선 방식은 패킷 교환 방식에서 연결형 서비스를 지원하기 위한 기능으로, 연결을 통해 전송되는 모든 패킷의 경로가 동일한 방식이다.
- 모든 패킷이 하나의 파이프(프로세스(또는 호스트, 네트워크)를 연결하는 논리적 통신 매체)로 표현되는 동일 경로로 전송되므로 패킷이 도착하는 순서는 보낸 순서와 같다.
##### 데이터 그램 방식(Datagram)
- 데이터 그램 방식은 패킷 교환 방식에서 비연결형 서비스를 이용해 패킷을 독립적으로 전송하는 방식을 말한다.
- 즉, 패킷을 송신하기 전에 미리 경로를 할당하지 않고 패킷들은 독립적인 경로로 전달된다.
- 각 전송 경로의 속도는 네트워크 혼잡도에 따라 가변적이여서, 패킷의 도착 순서는 보낸 순서와 다를 수 있다.
##### LAN(Local Area Network)
- LAN은 소규모 지역(단일 건물, 학교)에 위치하는 호스트로 구성된 네트워크이다.
- 규모가 아주 작을 때 단일 전송 케이블로 모든 호스트를 연결할 수 있다.
- LAN은 MAN이나 WAN 환경보다 호스트 간의 간격이 가깝기 때문에 데이터를 브로드 캐스팅(Broadcasting)방식(교환 필요 없음)으로 전송한다.
- 보통 수십Mbps ~ 수Gbps의 전송속도를 지원한다.
- 호스트를 연결하는 방식을 구성 형태에 따라 버스형, 링형으로 구분한다.
##### MAN(Metropolitan Area Network)
- MAN은 LAN보다 큰 지역(근처 여러 건물이나 한 도시)에 위치하는 호스트로 구성된 네트워크이다.
- 사용하는 하드웨어와 소프트웨어는 LAN과 비슷하다.(데이터를 브로드 캐스팅 방식으로 전송)
- MAN을 위한 국제 표준안은 DQDB(Distributed Queue Dual Bus)이며, 이 규격은 전송 방향이 다른 두 버스(단방향 선로)로 모든 호스트를 연결하는 구조를 지원한다.
- DQDB의 한 선로에서 헤드 역할을 하는 호스트가 일정 주기로 슬롯(Slot)을 내보내고, 이 슬롯은 꼬리 에 있는 호스트에서 처리를 완료할 때까지 계속 이동된다. 
- 이 과정에서 중간 호스트는 슬롯을 프레임으로 변형해 데이터를 전송할 수 있고, 다른 선로도 같은 원리가 적용되어, 데이터를 반대 방향으로 전송할 때 이용한다.
- 헤드 역할을 하는 호스트와 꼬리 역할을 하는 호스트의 물리적인 거리가 짧으면 새로운 연결을 구성할 수 있는데, 즉 헤드와 꼬리 호스트를 직접 연결하는 추가 전송 매체를 사용함으로써, 신뢰도를 높일 수 있다.
- DQDB의 특징은 다음과 같이 3가지로 요약할 수 있다.
1. 분산 데이터 큐를 유지한다.
2. 데이터 전송할 때 발생할 수 있는 충돌 문제를 해결하기 위해 슬롯 링 개념을 변형한 FIFO(First In First Out)기반의 공유 슬롯 방식을 사용한다.
3. ATM(Asynchronous Transfer Mode)과 호환이 가능하도록 53바이트의 프레임을 지원한다.
##### WAN(Wide Area Network)
- WAN은 국가 이상의 넓은 지역을 지원하는 네트워크 구조이다.
- 점대점(Point-to-Point)방식으로 연결되어 전송과 교환 기능이 반드시 필요하다.
- 전송 매체를 이용해 호스트를 일대일로 연결하는 방식으로 네트워크를 확장한다.
- 호스트간의 거리가 멀기 때문에 연결의 수가 증가할수록 전송 매체를 많이 사용하므로 비용이 많이 드므로, 총 연결 거리를 줄이면서 전체 호스트를 효율적으로 연결하도록 설계 해야한다.
- WAN에서는 호스트를 스타형, 트리형, 완전형, 불규칙형 등 다양한 구조로 연결할 수 있다.
##### PAN(Personal Area Network)
- 일반적으로 PAN은 Bluetooth나 UWB를 통한 연결망을 주로 의미한다.
- PAN은 의미론적으론 한 사람 주변에 형성된 컴퓨터 네트워크를 의미한다.
- 스마트폰과 연결된 Bluetooth 이어폰, 스마트워치(Bluetooth)가 대표적인 예이다.
#### LAN 환경에서 사용되는 호스트 연결방식
##### 버스형(Bus)
- 버스형은 공유 버스 하나에 여러 호스트를 직접 연결한다.
- 한 호스트가 전송한 데이터를 네트워크에 연결된 모든 호스트에 전송하므로, 브로드캐스팅 방식이다.
- 버스형에서는 전송 데이터가 모든 호스트에 전송되므로 라우팅 기능이 따로 필요 없다.
- 버스형의 작동방식은 목적지에 해당하는 호스트만 데이터를 내부 버퍼에 보관하고 나머지 호스트들은 데이터를 버림으로써 한 호스트만 데이터를 수신하는 방식이다.
- 이를 위해 각 호스트를 구분하는 호스트 주소를 사용하고, 전송 데이터에는 송수신 호스트의 주소를 표기한다.
- 둘 이상의 호스트가 데이터를 동시에 전송하려고 하면, 버스에서 데이터 충돌(Collision)이 발생할 수 있다.
- 이러한 데이터 충돌의 해결 방법은 충돌이 발생할 가능성 자체를 차단하는 사전 해결 방식과, 충돌을 허용하고 나중에 해결하는 사후 해결 방식이 있다.
- 대표적인 버스형 연결 형태인 이더넷(Ethernet)은 사후 해결 방식을 사용한다.
##### 링형(Ring)
- 링형은 전송 호스트의 연결이 순환 구조인 링 형태이다.
- 데이터는 시계나 반시계 방향으로 한 방향으로만 전송되며, 특정 호스트에서 전송한 데이터는 반드시 링을 한 바퀴 돌아 송신 호스트로 되돌아온다.
- 따라서, 연결된 모든 호스트가 전송 데이터를 수신하는 브로드캐스팅 방식을 지원한다.
- 또한 데이터를 송신한 호스트는 데이터를 네트워크에서 회수할 책임이 있다.
- 링형도 버스형과 마찬가지로 데이터 충돌이 생길 수 있으며, 일반적으로 토큰(Token)이라는 제어 프레임을 사용해 충돌 가능성을 차단한다.
- 데이터를 전송할 호스트는 전송용 토큰을 확보해야 하는데, 이 토큰은 네트워크에 하나만 존재하도록 설계되어 특정 시간에 데이터를 전송할 수 있는 호스트는 오로지 하나뿐이므로 데이터 충돌 가능성을 사전에 차단한다.
-  토큰은 연결된 호스트를 순환하도록 설계되어, 모든 호스트가 동등한 전송 기회를 갖는다.
##### 인터네트워킹(Internetworking)
- 인터네트워킹은 둘 이상의 서로 다른 네트워크를 연결하는 기능을 수행한다.
- 연결된 네트워크들은 물리적으로 같은 종류일 필요가 없으며, 상위 계층 프로토콜들이 지원하는 논리적 기능도 다를 수 있다.
- 따라서 인터네트워킹은 연결된 네트워크 간의 차이를 분석하여 전송 데이터를 적절히 중개할 수 있어야 한다.
- 이 네트워크 간의 차이는 일반적으로 연결형-비연결형 서비스, 프로토콜의 종류, 주소 표현 방법, 전송패킷의 크기, 멀티캐스팅-브로드캐스팅 지원 여부 등을 고려한다.
- 이러한 인터네트워킹 기능을 수행하는 시스템(장비)은 수행 기능에 따라 달라지는데, 일반적으로 리피터, 브리지, 라우터 등으로 구분된다.
##### 리피터(Repeater)
- 리피터는 물리계층의 기능을 지원한다.
- 물리적 신호는 전송 거리가 멀수록 감쇄되기 때문에, 한쪽에서 입력된 비트 신호를 단순히 증폭시켜 다른 쪽으로 전달하는 역할을 한다.
##### 브리지(Bridge)
- 브리지는 데이터 링크계층을 지원한다.(MAC 계층(2계층)을 지원하므로 물리 계층 기능도 수행)
- 한쪽에서 들어온 프레임의 MAC 계층 헤더를 다른 쪽의 MAC 계층 헤더로 변형해 전송할 수 있어, 종류가 다른 LAN을 연결시킬 수 있다.
- 브리지는 연결된 LAN 종류만큼 MAC 계층과 물리 계층을 해석할 수 있어야 한다.
- 브리지가 수신한 프레임의 목적지 주소와 송신 호스트의 주소가 같은 LAN에 소속되어 있으면 행동하지 않으며, 송수신 호스트 위치가 서로 다른 LAN에 속하면 중개 기능을 수행하는데, 이로 인해 브리지는 리피터보다 LAN 사이의 불필요한 트래픽 발생을 억제할 수 있다.
- 브리지의 양쪽 LAN이 같으면 프레임 헤더를 해석하는 간단한 작업으로 중개하고, 종류가 다르면 프레임 변환 등 복잡한 과정이 필요하다.
- 동작 방식에 따라 트랜스페런트 브리지와 소스 라우팅 브리지로 구분한다. 
##### 라우터(Router)
- 네트워크 계층 기능을 지원할 뿐만 아니라, 물리 계층, 데이터 링크 계층 기능을 수행한다.
- 교환 기능을 수행할 수 있으므로 여러 포트를 사용해 다수의 LAN을 연결하는 구조를 지원한다.
- 즉, 수신한 패킷을 해석해 적절한 경로로 전송하도록 경로를 배정하는 기능을 수행한다.
- 네트워크와 호스트에 대한 정보는 일반적으로 라우팅 테이블(Routing Table)에 보관된다.
##### 허브(Hub)
- 허브는 물리계층을 지원한다.
- 허브는 여러 대의 컴퓨터와 네트워크 장비를 연결하는 장치이며, 신호를 단순히 증폭시키는 리피터의 기능도 가지고 있다.
- 한 대의 허브를 중심으로 여러대의 컴퓨터와 네트워크 장비가 마치 별 모양으로 서로 연결되며, 같은 허브에 연결된 컴퓨터와 네트워크 장비는 모두 상호 간에 통신할 수 있게 된다.
##### 트랜스페런트 브리지(TransparentBridge)
- 라우팅 기능을 사용자에게 투명하게 보여준다.
- 필요한 라우팅 과정은 브리지가 자동으로 수행하므로, 브리지 사용자는 전송할 프레임의 주소부에 라우팅에 관한 정보를 추가하지 않아도 된다.
- 트랜스페런트 브리지는 공유 버스에서 구현되는 CSMA/CD(Carrier Sense Multiple Access/Collision Detection)방식과 토큰 버스 방식에서 사용한다. 사용자 입장에서 보면 간편하지만, 비효율적이다.
- 트랜스페런트 브리지가 제대로 동작하려면 라우팅 테이블(Routing Table) 정보가 정확해야 한다. 
##### 역방향 학습(Backward Learning)
- 네트워크 동작 과정에서 라우팅 정보를 얻는 방식을 역방향 학습 알고리즘이라고 한다.
- 트랜스패런트 브리지에서 소개한 라우팅 테이블은 LAN이 동작하면서 자동으로 생성되는데, 초기에는 라우팅 테이블이 비어있어 프레임의 수신자가 어느 쪽 포트에 위치하는지 판단할 수가 없다.
- 이 때 플러딩(Flooding) 알고리즘을 사용하여 입력된 프레임을 브리지의 모든 포트 방향으로 전달한다.
- 이 플러딩 알고리즘의 동작 과정에서 입력된 송신 호스트의 주소로 송신 호스트가 몇 번 포트에 연결되었는지 알 수 있다.
- 이와 같은 데이터 전달 과정에서 프레임의 송신 호스트의 주소와 포트 번호의 정보를 알아내어 라우팅 테이블에 반영한다.
- 이 방법을 사용하면, 시간이 경과함에 따라 라우팅 테이블의 정보가 계속 누적되고 라우팅 정보를 효과적으로 수집할 수 있다.
- 즉, 이처럼 네트워크 동작 과정에서 라우팅 정보를 얻는 방식을 역방향 학습 알고리즘이라고 한다.
- 네트워크에 이중 경로가 존재하면 잘못된 라우팅 정보를 얻을 수 있으므로 이중 경로가 존재하지 않도록 네트워크를 설계해야 한다.
##### 스패닝 트리(SpanningTree)
- 스패닝 트리는 네트워크의 비순환 구조를 말하며, 이를 지원하는 알고리즘을 스패닝 트리 알고리즘이라 한다.
- 네트워크의 순환 구조로 올바르게 작동하지 않는 역방향 학습 알고리즘이 올바르게 작동하게 스패닝 트리를 사용한다.
##### 스패닝 트리 구성 방법
1. 스패닝 트리를 구성하기 위해, 먼저 임의의 브리지를 트리 구조의 최상위 브리지인 루트로 지정해야 한다.
2. 브리지가 자신의 고유 번호를 서로 공개함으로써, 번호가 가장 낮은 브리지를 루트로 선정할 수 있다.
3. 루트 브리지에서 다른 모든 브리지까지의 최단 경로 트리를 구성하는 방식으로 LAN을 구축하는 과정을 진행한다.
##### 소스 라우팅 브리지(Source Routing Bridge)
- 소스 라우팅 브리지는 일반적으로 링 구조(순환 구조)의 네트워크에서 사용한다.
- 트랜스페런트 브리지와 달리 프레임이 수신 호스트까지 도달하기 위한 라우팅 정보를 송신 호스트가 제공한다.
- 다시 말해서, 송신 프레임 내부에 수신 호스트까지 도달하기 위한 모든 경로를 기술함으로써, 중간 브리지에 필요한 라우팅 정보가 프레임 자체에 포함된다.
- 프레임을 수신한 브리지는 이 정보를 이용해 프레임을 적절한 경로로 전달한다.
##### 고정 경로 배정(Static(Fixed) Routing)
- 고정 경로 배정은 송수신 호스트 사이에 영구불변의 경로를 배정한다.
- 라우팅 테이블의 경로 정보 변경은 네트워크 구성이 변경된 경우에만 가능하다.
- 단점으로 전송 경로가 고정되므로 트래픽 변화에 따른 동적 경로 배정이 불가능하다.
- 따라서, 송수신 호스트 사이의 트래픽을 미리 측정하여 고정 경로를 적절히 배정시킨다면 간단한 구현만으로도 효과적인 라우팅이 가능하다.
##### 적응 경로 배정(Dynamic(Adaptive) Routing)
- 적응 경로 배정은 인터넷에서 사용되는 라우터가 사용하며, 인터넷 연결 상태가 변하면 이를 데이터 그램의 전달 경로에 반영한다.
- 경로 배정에 영향을 주는 요소는 특정 네트워크나 라우터가 정상적으로 동작하지 않을 때나, 네트워크의 특정 위치에서 혼잡이 발생한 경우이다.
- 특정 위치에서 어떤 변화가 발생했을 때 이 변화를 네트워크의 모든 라우터에 동시에 반영하는 것은 현실적으로 불가능하므로, 라우터 간의 정보의 불일치성(Inconsistency)이 발생할 수 있다.
- 이러한 불일치성을 해소하기 위해 정보 교환이 빠르게 자주 이루어져야 하는데, 이 또한 네트워크의 트래픽과 라우터의 처리 부담을 증가시킨다.
##### 자율 시스템 (AS(Autonomous System))
- 자율 시스템은 다수의 라우터로 구성할 수 있고, 라우터들은 공통의 라우팅 프로토콜을 사용하여 정보를 교환한다.
- 즉, 자율 시스템은 동일한 라우팅 특성에 의해 동작하는 논리적인 단일 구성체라고 볼 수 있다.
- 자율 시스템 내부에서 사용하는 공통 프로토콜을 내부 라우팅 프로토콜, 자율 시스템들 간에 사용하는 라우팅 프로토콜을 외부 라우팅 프로토콜이라 한다.
##### 내부 라우팅(Interior Routing(RIP, OSPF))
- 자율 시스템 내부에서 사용하는 공통 프로토콜을 내부 라우팅 프로토콜이라 하며, 라우터들끼리 라우팅 정보를 교환하는 용도로 사용한다.
- 내부 라우팅 프로토콜에는 RIP, OSPF등이 있다.
##### 라우팅 정보 프로토콜(RIP: Routing Information Protocol)
- 라우팅 정보 프로토콜은 UDP/IP 상에서 동작하는 라우팅 프로토콜이다.
- 라우팅 정보 프로토콜은 경유할 가능성이 있는 라우터를 홉수로 수치화하여, DVA(Distance Vector Algorithm)라는 알고리즘으로 인접 호스트와의 경로를 동적으로 교환하는 일이다. 
- 패킷이 목적 네트워크 주소에 도착할 때까지의 최단 경로를 결정한다. 
##### 최단 경로 우선 프로토콜(OSPF: Open Shortest Path First)
- 인터넷 프로토콜(IP) 네트워크를 위한 링크 스테이트 라우팅 프로토콜이다. 
- 링크 스테이트 라우팅 알고리즘을 사용하며, 하나의 자율 시스템에서 동작하면서 내부 라우팅 프로토콜의 그룹에 도달한다.
##### 외부 라우팅(Exterior Routing(BGP))
- 자율 시스템들 간에 사용하는 라우팅 프로토콜을 외부 라우팅 프로토콜이라 한다.
- 외부 라우팅 프로토콜에는 BGP 등이 있다.
##### 경계 경로 프로토콜(BGP: Border Gateway Protocol)
- BGP는 외부 라우팅 프로토콜로 자율 시스템간 사용되는 라우팅 프로토콜이다.
- 인터넷에서 주 경로 지정을 담당하는 프로토콜의 한 종류이다.
- 인터넷에서 목적지까지 경유하는 자율 시스템중 라우팅 및 자율 시스템의 순서를 전송하기 위해 설계된 경로 지정 알고리즘으로, 표준화된 외부 게이트웨이 프로토콜의 하나이다.
##### QoS(Quality of Service)
- QoS는 인터넷 환경에서 전송 서비스 문제를 다루는 중요한 고려 대상 중 하나이다.
- QoS는 보통 연결형 서비스를 위한 것이지만, 비연결형 서비스에도 부분적으로 적용된다.
- QoS는 주로 전송 계층 사용자가 요청하므로, 전송 계층 연결을 설정할 때 필요한 서비스의 정도를 매개변수로 표시한다.
- QoS에서는 전송데이터를 특징에 따라 여러 종류로 분류한다.
- 영상데이터는 대용량 실시간 전송이 필요하지만, 전송 오류문제에는 상대적으로 관대하며, 일반 컴퓨터 데이터는 실시간 기능은 필요 없지만, 전송 오류에는 매우 민감하다. 
- IP프로토콜은 우선순위를 조절하는 기능이 없으므로, 모든 패킷을 동일한 기준으로 처리하는 단점이 있다.
##### 자주 언급되는 QoS 매개변수
##### 연결 설정(해제) 지연(Connection Establishment Delay)
- 연결 설정을 위한 request 프리미티브 발생과 confirm 프리미티브 도착 사이의 경과 시간이다.
- 경과 시간이 짧을수록 서비스 품질이 좋다고 할 수 있으며, 네트워크 혼잡도의 영향을 많이 받는다.
- 연결 해제 시 동일한 기준을 적용할 수 있다.
##### 연결 설정(해제) 실패 확률(Connection Establishment Failure Probability)
- 임의의 최대 연결 설정 지연 시간을 기준으로 연결 설정이 이루어지지 않을 확률이다.
- 연결 해제 시 동일한 기준을 적용할 수 있다.
##### 전송률(Throughput)
- 초당 전송할 수 있는 바이트 수이다.
- 전송률은 양방향 값이 다를 수 있으므로, 별개로 다뤄야 한다.
##### 전송지연(Transit Delay)
- 송신 호스트가 전송한 데이터가 수신 호스트에 도착할 때까지 경과한 시간이다.
- 전송률처럼 양방향을 별개로 다뤄야 한다.
##### 전송 오류율(Residual Error Rate)
- 임의의 시간 구간에서 전송된 총 데이터 수와 오류 발생 데이터 수의 비율이다.
##### 우선순위(Priority)
- 우선순위가 높은 연결이 우선순위가 낮은 연결보다 먼저 처리되며, 좋은 서비스를 제공 받는 것을 의미한다.
